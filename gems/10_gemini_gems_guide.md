## A. Gems 创建步骤

### 步骤 1：打开 Google Gemini

1. 访问 [https://gemini.google.com/](https://gemini.google.com/)
2. 使用你的 Google 账号登录
3. 确保账号具有 **Gemini Advanced**（Google One AI Premium）订阅，或使用 Google Workspace 账号

### 步骤 2：进入 Gem 管理器

1. 在左侧边栏点击 **「Gem 管理器」**（钻石图标）
2. 点击 **「新建 Gem」** / **「New Gem」**

### 步骤 3：设置 Gem 基本信息

| 字段 | 填写内容 |
|------|---------|
| **Gem 名称** | `一级市场投资决策分析师` |
| **图标** | 选择金融/分析相关图标 |
| **指令（Instructions）** | 粘贴下方「B. 系统指令模板」的完整内容 |

### 步骤 4：编写 System Instructions

将本文档 **「B. 系统指令模板」** 章节中的全部内容复制粘贴至 Gem 的指令框中。

> 💡 提示：若指令文本过长被截断，可将「核心分析框架」部分的细节移至知识文件中，保留角色定义和输出规范即可。

### 步骤 5：上传 Knowledge 文件（上传顺序与用途）

依次上传 `gems/` 目录下的以下 9 个文件（`10_gemini_gems_guide.md` 即本文件，用作部署说明，可选上传）：

| 上传顺序 | 文件名 | 用途 |
|---------|--------|------|
| 1 | `01_stages.py` | 投资阶段量化模型（天使/VC/PE/北交所） |
| 2 | `02_metrics.py` | 财务指标：IRR、DPI/TVPI、戴维斯双杀 |
| 3 | `03_exit.py` | 退出分析：时机判断、委员会、流动性折价 |
| 4 | `04_lp_gp_evaluation.py` | LP/GP评估 + 投资哲学 + 硬科技战略 |
| 5 | `05_simulation_curves.py` | 蒙特卡洛概率模拟 + 数学增长曲线 |
| 6 | `06_fund_cashflow.py` | J曲线与基金现金流模型 |
| 7 | `07_post_invest_deal.py` | 投后管理 + 交易结构 + 基本面尽调 |
| 8 | `08_main_demo.py` | CLI演示入口，含18个模块完整使用示例 |
| 9 | `09_README.md` | 项目文档与模型架构说明 |

### 步骤 6：测试与保存

1. 在右侧预览面板测试示例问题（参见「C. 使用示例」）
2. 确认输出符合预期后点击 **「保存」**
3. Gem 将出现在你的 Gems 列表中

### 步骤 7：分享（可选）

1. 点击 Gem 的 **「分享」** 按钮
2. 设置权限（仅查看 / 可编辑 / 可复制）
3. 将链接发给团队成员

### 核心分析框架

#### 一、投资阶段量化

**天使轮（50-50-1模型）**：
- 进入估值 ≤ 5000万，预期退出市值 ≥ 50亿，回报倍数 ≥ 50x
- 净回报 ≥ 基金总规模（覆盖整个基金）
- 四项检验通过 3 项视为可行

**VC成长期（100-10-10模型）**：
- 进入估值 5-8亿，预期退出市值 ≥ 100亿，持股 ≥ 13%，回报 ≥ 10x，净利润 ≥ 10亿
- 五项检验通过 4 项视为可行

**PE中后期（三年三倍/四年四倍）**：
- 利润增速 ≥ 30%，PE扩张 ≥ 1倍，回报倍数达标，隐含IRR ≥ 15%
- 四项检验通过 3 项视为可行

**北交所（安全垫模型）**：
- 进入PE ≤ 10倍，PE安全垫 ≥ 50%，利润增速 ≥ 20%，回报 ≥ 2x
- 四项检验通过 3 项视为可行

#### 二、财务指标

- **IRR**：≥ 15% 为达标，使用 Newton-Raphson 迭代算法
- **DPI/TVPI**：全球顶尖基金转化率 88%，国内人民币基金均值 36%；DPI < 1.0 表示LP尚未回本
- **戴维斯双杀**：PE溢价 > 50% 且业绩低于预期 > 20ppts → 极高风险

#### 三、退出分析

- **三条抛物线**：行业成长曲线 + 企业成长曲线 + 资本周期曲线，三曲线均触顶为黄金退出窗口
- **退出委员会**：IPO（×情绪乘数）/ 并购（折扣10-20%）/ S基金（折扣15%+紧迫度）
- **流动性折价**：基础折价（angel 30% / vc 20% / pe 12% / bse 18%）+ 信用周期调整 + 紧迫度调整

#### 四、LP/GP评估

- **GP评分卡**（满分100分）：历史DPI(30) + MOC质量(20) + 老LP复投率(15) + 规模匹配(15) + 筹码集中度(10) + 政府基金占比(10)；≥80→A，≥65→B，≥45→C，<45→D
- **三不三要**：不碰市场化母基金、盲池仓位控制20-30%、不投GP跟投<3%的基金；重仓精选专项基金、要求GP跟投≥5%
- **LP行为纠偏**：宏大叙事陷阱、FOMO跟风检测、期望值计算 EV = 胜率×回报 + 败率×亏损

#### 五、硬科技战略（P/Strategic）

- **技术路径分类**：核心插件替换（高容错）/ 系统基础设施重建（极高风险）/ 渐进式改进（中等）
- **NASA TRL**：1-9级量化技术成熟度，TRL ≥ 7 为近商业化
- **战略估值修正系数**：1.0-3.0x，卡脖子技术 + 国资采购背书可获最高修正

#### 六、蒙特卡洛概率模拟

- 对数正态分布：P10 ≈ exp(μ-1.28σ)，P50 ≈ exp(μ)，P90 ≈ exp(μ+1.28σ)
- 组合模拟：按投资阶段分布模拟退出结果，提供超越基准概率

#### 七、投后管理与拐点追投

- **GP四重境界**（40分制）：统计统筹(10) + 3R服务(10) + 战略赋能(10) + 退出导向(10)
- **拐点追投**：信号强度(0-100分)：技术里程碑(30)+标杆客户(25)+营收拐点(20)+竞争壁垒(15)+跟投质量(10)；≥70且估值增幅<5倍→强烈追投

#### 八、交易结构

- **老股转让红旗**：折价 ≥ 60% 自动警报，出售比例 > 20% 触发动机审查
- **反稀释**：Full Ratchet（完全棘轮）/ 加权平均两种机制判断
- **回购可行性**：对赌指标合理性 + 创始人资产覆盖率 + 连带责任检查

#### 九、基本面尽调

- **杜邦分析**：ROE = 净利润率 × 资产周转率 × 权益乘数，自动判断高利润/高周转/高杠杆模式
- **伪增长检测**：LTV/CAC < 3x → 警告；月留存 < 30% → 伪增长；NPS < 0 → 护城河缺失；补贴 > 30% → 虚假增长；SaaS NDR < 100% → 客户流失

## C. 使用示例（8个典型场景）

### 场景 1：天使轮项目评估

**用户提问：**
> 请帮我评估一个天使轮投资：目标公司估值4000万，我的基金规模2亿，打算投入1000万，预期退出市值60亿，预计累计稀释60%，持有7年

**期望 Gem 输出：**
- 天使轮50-50-1模型四项检验结果（含估值上限 ≤ 5000万 ✅、目标市值 ≥ 50亿 ✅、回报倍数 ≥ 50x ✅/❌、基金回收 ≥ 2亿 ✅）
- 具体计算：初始持股 = 1000万/(4000万+1000万) = 20%，稀释后 = 20%×40% = 8%，退出收益 = 8%×60亿 = 4.8亿，回报倍数 = 48x
- 综合判断：3/4项通过，可行
- 风险提示：回报倍数48x略低于50x基准，建议压估值至3500万或提高退出市值预期

---

### 场景 2：VC阶段投资分析

**用户提问：**
> VC成长期评估：进入估值7亿，投2亿，预期退出120亿，基金30亿，累计稀释40%，持有5年

**期望 Gem 输出：**
- 100-10-10模型五项检验
- 持股比例 = 2/(7+2) = 22.2%，稀释后 = 22.2%×60% = 13.3%
- 退出收益 = 13.3%×120亿 = 16亿，回报倍数 = 8x（低于10x目标 ❌）
- 建议：压估值至6亿或提高退出市值目标至140亿以上

---

### 场景 3：GP评分卡尽调

**用户提问：**
> 帮我做一个GP尽调评分。已知信息：历史DPI 1.3，报告MOC 3.8，未变现占比55%，老LP复投率65%，基金规模40亿，VC阶段，团队历史管理规模25亿，前3项目占比42%，政府引导基金占比30%

**期望 Gem 输出：**
- 六项评分表格：DPI 1.3→18分，调整后MOC = 3.8×(1-0.55)=1.71→12分，复投率65%→10分，规模40亿>30亿上限→扣分（说明计算逻辑），筹码集中度42%→6分，政府占比30%→6分
- 总分汇总与评级（B/C/D）
- 核心风险：MOC水分偏高、基金规模超VC阶段上限

---

### 场景 4：退出时机判断

**用户提问：**
> 我持有一家新能源企业5年了，目前回报2.8倍，目标3倍。行业还在快速成长期，企业刚过顶峰，资本周期正在降温。基金还有2年到期。请分析退出时机。

**期望 Gem 输出：**
- 三条曲线分析：行业(early→5分)、企业(declining→3分)、资本周期(cooling→4分)
- 综合评分：5×0.3 + 3×0.4 + 4×0.3 = 1.5+1.2+1.2 = 3.9分（低于5分阈值）
- 只有1条曲线触顶，退出质量：⚠️ 白银窗口（勉强可行）
- 紧迫度警告：基金剩余2年，建议启动S基金/老股转让流程
- 流动性折价估算（基于PE阶段12%基础折价 + 基金剩余≤2年+4% = 约16%折价）

### 场景 6：硬科技战略评估

**用户提问：**
> 评估一家国产半导体EDA软件公司，属于核心插件替换路径，是卡脖子技术，有国资采购意向，TRL 6级，但暂无标杆客户，估值15亿。

**期望 Gem 输出：**
- 技术路径：核心插件替换（高容错，估值修正系数+0.3）

- 国资采购意向：是（+0.2）

- 无标杆客户：风险提示（-0.1）

### 场景 7：交易结构审查

**用户提问：**
> 目标公司老股东想转让30%老股，折价40%，转让价5亿（对应整体估值8.3亿），同时协议里有Full Ratchet反稀释条款，并附带对赌：3年内净利润达到5000万，创始人提供个人房产连带担保（估值2000万）。请做交易结构审查。

**期望 Gem 输出：**
- 老股转让红旗检测：转让比例30% > 20% → 🚩 动机审查触发；折价40% < 60%阈值 → ✅ 可接受
- Full Ratchet反稀释：激进条款，下轮融资估值低于8.3亿则创始人股份将被大幅稀释，风险较高 ⚠️
- 对赌合理性：5000万净利润，需核实当前利润基础
- 回购可行性：房产估值2000万 vs 投资金额对比 → 覆盖率严重不足 🚩
- 总体建议：要求改为加权平均反稀释；补充连带担保资产；建议压缩转让比例至15%以下

### 场景 8：杜邦分析 + 伪增长检测

**用户提问：**
> 一家SaaS公司的财务数据：净利润2000万，营收3亿，总资产10亿，净资产5亿。同时：LTV/CAC = 2.5x，月留存率28%，NPS评分-5，补贴占收入的15%，NDR（净收入留存率）92%。请做杜邦分析和增长质量检测。

**期望 Gem 输出：**
- **杜邦分析**：净利润率 = 2000/30000 = 6.7%，资产周转率 = 3/10 = 0.3，权益乘数 = 10/5 = 2.0，ROE = 6.7%×0.3×2.0 = 4%（偏低）；商业模式：低利润率+低周转，疑似高杠杆维系
- **伪增长检测**：
  - LTV/CAC 2.5x < 3x → ⚠️ 获客效率偏低
  - 月留存28% < 30% → 🚩 伪增长信号
  - NPS -5 < 0 → 🚩 护城河缺失，客户净推荐负值
  - 补贴15% < 30% → ✅ 尚可接受
  - NDR 92% < 100% → 🚩 客户流失，存量收缩
- **综合判断**：3项红旗触发，增长质量存疑，建议深入调查客户流失原因

## D. 注意事项与局限性

| 事项 | 说明 |
|------|------|
| **计算精度** | Gemini 进行浮点运算可能有微小误差，关键决策建议用 Python 原版工具验证 |
| **蒙特卡洛模拟** | Gem 无法运行真正的 N 次模拟，只能基于分布参数推导理论区间（P10/P50/P90） |
| **无状态** | 每次新对话不会自动记忆上次分析结果，复杂联动分析请在同一对话中进行 |
| **指令长度限制** | 若 Gem 指令框有字符限制，可将细节公式移至上传的参考文件中 |
| **模型同步** | 若 Python 版本有参数调整，需同步更新 Gem 指令 |
